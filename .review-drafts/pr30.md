## 🔴 Required (CRITICAL): `requireManager()` missing on the edit page

`app/(dashboard)/employees/[id]/edit/page.tsx` doesn't call `requireManager()` before `createClient()`. RLS blocks the DB access so no data leaks, but unauthenticated users flow into the `notFound()` path instead of being redirected to `/login`. This breaks the project's auth pattern.

```ts
// app/(dashboard)/employees/[id]/edit/page.tsx
import { requireManager } from '@/lib/auth/manager'

export default async function EditEmployeePage({ params }) {
  await requireManager() // ← add
  const { id } = await params
  const supabase = await createClient()
  // ...
}
```

Please add it to `new/page.tsx` as well (form submission is protected on the Action side, but the policy is defense-in-depth at every layer).

## 🔴 Required: archive / restore swallow DB errors

`archiveEmployee` / `restoreEmployee` discard the `error` from `update()`, so RLS rejections or network errors silently show as success. Please check errors using the same pattern as `createEmployee` / `updateEmployee`:

```ts
export async function archiveEmployee(id: string): Promise<void> {
  await requireManager()
  const supabase = await createClient()
  const { error } = await supabase.from('employees').update({ is_active: false }).eq('id', id)
  if (error) throw new Error('Failed to archive employee')
  revalidatePath('/employees')
}
```

Since these run through simple forms that don't use `useActionState`, letting `throw` flow to Next.js's error boundary is the easy route.

## 🟡 Recommended: full phone numbers shown in the list

CLAUDE.md says phone numbers are sensitive personal data, so masking to the last 4 digits in the list (e.g. `+1 *** *** *4567`) and showing the full number only on the edit screen would be safer. It's an internal admin tool, but considering screen-share accidents, raw display in the list is worth avoiding.
