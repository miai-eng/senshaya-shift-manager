## 🟡 推奨: 電話番号バリデーション緩和の影響

`/^\+1[2-9]\d{2}[2-9]\d{6}$/` から `/^\+1\d{10}$/` への変更で、`+10000000000` のような不正値が通るようになりました。スペース除去のためならバリデーションロジックを変えるよりも、`parseFormData` で先にスペースを除去して、バリデーション自体は厳格な NANP regex のままにする方が安全です:

```ts
// parseFormData 側
const phone = String(formData.get('phone') ?? '').replace(/\s+/g, '')
// バリデーション側はそのまま厳格に
const CANADIAN_PHONE_RE = /^\+1[2-9]\d{2}[2-9]\d{6}$/
```

ビザ留学生で実電話番号は使うので、本物の番号しか弾けないのが望ましいです。
