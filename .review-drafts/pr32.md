## 🟡 Recommended: impact of the relaxed phone validation

Changing `/^\+1[2-9]\d{2}[2-9]\d{6}$/` to `/^\+1\d{10}$/` now allows invalid values like `+10000000000`. If the goal was space removal, it's safer to strip spaces in `parseFormData` first and keep the strict NANP regex for validation:

```ts
// in parseFormData
const phone = String(formData.get('phone') ?? '').replace(/\s+/g, '')
// keep validation strict
const CANADIAN_PHONE_RE = /^\+1[2-9]\d{2}[2-9]\d{6}$/
```

Working-visa students use real phone numbers, so ideally only genuine numbers pass.
