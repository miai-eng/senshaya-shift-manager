Thanks for the recurring days off feature. The weekday mapping (0=Sun ... 6=Sat) matches Postgres `EXTRACT(DOW)`, and the edit-screen pre-population works correctly — nice work.

One data-integrity concern to address:

## 🔴 Required: `delete → insert` is not atomic

The pattern below inside `updateEmployee` isn't wrapped in a transaction, so a successful delete followed by a failed insert **wipes out all of that employee's recurring days off**:

```ts
await supabase.from('recurring_days_off').delete().eq('employee_id', id)
// ← a failure here means data loss
if (offDays.length > 0) {
  await supabase.from('recurring_days_off').insert(...)
}
```

Running a transaction directly from Supabase JS is difficult, so two options:

**Option A: diff-based update (apply additions and removals separately)**

```ts
const oldSet = new Set(existing.map((r) => r.day_of_week))
const newSet = new Set(offDays)
const toAdd = [...newSet].filter((d) => !oldSet.has(d))
const toRemove = [...oldSet].filter((d) => !newSet.has(d))

if (toRemove.length > 0) {
  await supabase
    .from('recurring_days_off')
    .delete()
    .eq('employee_id', id)
    .in('day_of_week', toRemove)
}
if (toAdd.length > 0) {
  await supabase
    .from('recurring_days_off')
    .insert(toAdd.map((d) => ({ employee_id: id, day_of_week: d })))
}
```

**Option B: a Postgres RPC function**
Create `update_recurring_days_off(employee_id, days)` as a SQL function in the Supabase dashboard, with BEGIN/COMMIT inside. Requires a migration too.

For the MVP stage, **Option A** is simple and sufficient in my view.

## 🟡 Recommended: range checking in `parseOffDays`

```ts
function parseOffDays(formData: FormData): number[] {
  return formData.getAll('day_of_week').map((v) => parseInt(v as string, 10))
}
```

`NaN` and out-of-range values slip through. The DB CHECK constraint catches them eventually, but rejecting earlier gives clearer error messages:

```ts
return formData
  .getAll('day_of_week')
  .map((v) => parseInt(v as string, 10))
  .filter((n) => Number.isInteger(n) && n >= 0 && n <= 6)
```
