定期休み機能ありがとうございます。曜日マッピング（0=日 ... 6=土）が Postgres `EXTRACT(DOW)` と一致していて、編集画面の pre-population も正しく動いていて良いと思いました。

データ整合性の観点で対応お願いします:

## 🔴 必須: `delete → insert` がアトミックでない

`updateEmployee` 内の以下のパターン、トランザクションで囲まれていないため、delete 成功 / insert 失敗で **その従業員の定期休みが全消失する** リスクがあります:

```ts
await supabase.from('recurring_days_off').delete().eq('employee_id', id)
// ← ここで失敗するとデータロス
if (offDays.length > 0) {
  await supabase.from('recurring_days_off').insert(...)
}
```

Supabase JS から直接トランザクションを張るのは難しいので、対策案2つ:

**案A: 差分更新にする（追加・削除を分けて適用）**

```ts
const oldSet = new Set(existing.map((r) => r.day_of_week))
const newSet = new Set(offDays)
const toAdd = [...newSet].filter((d) => !oldSet.has(d))
const toRemove = [...oldSet].filter((d) => !newSet.has(d))

if (toRemove.length > 0) {
  await supabase
    .from('recurring_days_off')
    .delete()
    .eq('employee_id', id)
    .in('day_of_week', toRemove)
}
if (toAdd.length > 0) {
  await supabase
    .from('recurring_days_off')
    .insert(toAdd.map((d) => ({ employee_id: id, day_of_week: d })))
}
```

**案B: Postgres RPC関数にする**
Supabase ダッシュボードで `update_recurring_days_off(employee_id, days)` を SQL 関数として作り、内部で BEGIN/COMMIT する。マイグレーションも追加が必要。

MVP段階だと **案A** が簡単で十分かなと思います。

## 🟡 推奨: `parseOffDays` の範囲チェック

```ts
function parseOffDays(formData: FormData): number[] {
  return formData.getAll('day_of_week').map((v) => parseInt(v as string, 10))
}
```

`NaN` や 0-6 範囲外の値も通ってしまいます。DB の CHECK 制約で最終的に弾かれますが、手前で弾いた方がエラーメッセージが分かりやすいです:

```ts
return formData
  .getAll('day_of_week')
  .map((v) => parseInt(v as string, 10))
  .filter((n) => Number.isInteger(n) && n >= 0 && n <= 6)
```
